Purpose

This memo outlines the steps to protect Microsoft Word documents while allowing clients to fill in specific fields. This ensures that essential document structures remain unchanged while allowing clients to provide necessary information.

Instructions for Protecting a Word Document with Fillable Fields

Step 1: Enable the Developer Tab

To create fillable fields, you must first enable the "Developer" tab in Microsoft Word:

  1. Open Microsoft Word.
  2. Click File > Options. (This may require selecting More at the bottom of the file options)
  3. Select Customize Ribbon on the left panel.
  4. In the right panel under "Customize the Ribbon," check the box for Developer.
  5. Click OK.

Step 2: Insert Fillable Form Fields

Once the Developer tab is enabled:

  1. Click the Developer tab on the ribbon.
  2. Select Design Mode (this allows you to insert and edit form controls).
  3.  
  4. Click Rich Text Content Control or Plain Text Content Control, depending on your needs:
    • Rich Text: Allows formatting (bold, italics, etc.).
    • Plain Text: Only allows unformatted text.
  5. Click where you want to insert the field and then click the respective button.
  6. You may rename the field for clarity by selecting the inserted control, then clicking Properties in the Developer tab.

Step 3: Protect the Document to Restrict Editing

To ensure only the form fields can be edited:

  1. Click Developer > Restrict Editing.
  2. In the "Restrict Editing" pane on the right:
    • Check Allow only this type of editing in the document.

o   Select Filling in forms from the dropdown menu.

  1. Click Yes, Start Enforcing Protection.
  2. If prompted, enter a password to prevent unauthorized changes (optional but recommended).

Step 4: Save and Distribute the Document

  1. Click File > Save As.
  2. Choose a location and format (keep it as a .docx file for best results).
  3. Distribute the document to clients for completion.

Clients will only be able to fill in the designated fields while the rest of the document remains locked.

Conclusion
